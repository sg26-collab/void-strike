# Project Guidelines

## Git & GitHub

After every meaningful unit of work, commit changes and push to GitHub:

- Use clean, descriptive commit messages that explain **what** changed and **why**
- Commit format: short summary line (≤72 chars), then a blank line, then bullet details if needed
- Always push after committing — `git push origin main` — so work is never only local
- Never batch unrelated changes into a single commit; keep commits focused
